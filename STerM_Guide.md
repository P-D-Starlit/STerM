# STerM 사용설명서

- STerM(a.k.a. 뜨따뜨따)은 매크로 기반으로 작동하는 자동화 솔루션 프로그램입니다.
- PuTTY와 Tera Term에 비해 매크로 의존도가 높지만, 잘 사용하면 자동화 업무 개발을 손쉽게 할 수 있습니다.
- STerM은 Starlit Terminal을 줄인 것으로 COMx 계열 터미널에 최적화된 매크로 전용 프로그램이라는 의미에서 STerM의 형태로 맨 끝 글자를 대문자로 표시합니다. 즉, 맨 끝 M은 Macro를 의미합니다.
- 파일 형식은 STerM을 줄인 `.stm`을 사용하며, 매크로는 내부 Starlit Compiler에 의해 `.stb`로 변환되어 실행됩니다.
  - ~~ST Microelectronics의 그 stm과는 다릅니다. 우연히 Starlit Terminal Macro를 줄이다 보니 stm이 되었을 뿐입니다.~~

## 제품 구성

- STerM은 기본적으로 Console 창 인터페이스를 제공합니다.
- STerM은 COM 포트를 통해 주변 장치와의 통신을 제공합니다.
- STerM은 Starlit 언어 기반의 가상 머신으로 작동하는 TTY 및 COMx 제어 프로그램으로, COMx를 사용하여 제어 가능한 주변 장치를 자동화 컨트롤하여 시간이 많이 걸리는 실험 시간을 단축할 수 있도록 설계 하였습니다.

## 업무 효율화/자동화

- 효율적인 업무를 위하여 다중 스타일 지향 언어인 Starlit 언어 사용하되, 아래 설명서에는 Python 스타일에 맞춘 권장 코딩 스타일을 제공하며, 업무 진행 시 아래에서 제공되는 코딩 스타일에 준수하시기 바랍니다.
  - 코딩 스타일 미준수하여 작성하는 경우 추후 코드 공유 등 업무에 있어서 효율성이 하락될 수 있습니다. 일관된 문법을 사용함으로써 여러 사람이 공동으로 쉽게 업무 목표에 달성할 수 있도록 하는 것을 권장합니다.
  - 주석 작성은 `# `을 사용합니다. `#` 입력 후 **한 칸 띄운** 다음 주석을 작성해야 하며, 띄어쓰기 없이 작성하면 컴파일 오류가 발생할 수 있으며, 띄어쓰기 2칸 이상 사용은 가독성 문제가 발생하여 권장하지 않습니다.
  - Starlit 언어에서 Printf 계열 함수를 지원하는 것과 다르게 STerM은 Printf 계열 함수 사용을 권장하지 않습니다. Python 방식의 f-string 문자열 포매팅 사용을 권장합니다. 자세한 내용은 Python에서 제공하는 f-string 관련 내용 참고 바랍니다.

## Console 창 제어

### 입출력 예제

- 콘솔 창에 출력하기 위해 `CONSOLE` 객체를 사용할 수 있습니다.
  ```Py
  # Print Example
  
  main:
      CONSOLE << "/rHello, /gWorld!" << CONSOLE.ln
      WaitKey
  ```

- 콘솔 창에 값을 입력받기 위해 아래와 같이 작성할 수 있습니다.
  ```Py
  main:
      a = int                                # define Variable a.
      "Input : " >> CONSOLE >> a             # Print "Input : " and input a
      CONSOLE << f"The Input Value is {a}."  # Print Result.
      WaitKey
  ```

- STerM은 f-string 포매팅을 지원합니다.
- WaitKey를 사용하여 사람이 키보드를 입력할 때까지 기다릴 수 있습니다.
  - main함수 종료되면 창이 꺼지므로 main함수 종료 전 결과 확인을 위해 WaitKey가 마지막에 있는지 확인 부탁드립니다.

- STerM은 `int`형과 `float`형 변수를 지원합니다. 문자열 자체는 최대 255자까지 `str`형 변수로 사용할 수 있습니다.

### 글자 위치 예제

- STerM은 자동화 테스트 현황을 쉽게 확인할 수 있도록 글자 서식 및 위치 조절이 가능합니다.
- Console 창에서 이 기능을 적극적으로 사용하면 현재 실험 상황을 쉽게 확인할 수 있습니다.
- 아래 예제는 loop가 돌아간 횟수를 출력하는 코드입니다. 1초에 cnt값을 1 증가합니다.

  ```Py
  cnt = 0

  main:
      for i:0:20:
          CONSOLE << f"/0CYCLE = {i:02d}"
          Delay 1000
      WaitKey
  ```

- `/0` : 글자 위치를 0번째 줄에 놓습니다.
  - `/0`~`/9` : 글자 위치를 0~9번째 줄에 놓습니다.
  - `/10`이라고 작성하면 10번째 줄에 출력되는게 아니라 *1번째 줄에 맞춰놓고 0이 한 번 더 출력됩니다*. 따라서 10번째 줄에 출력하기 위해서는 `/10`아 아닌 `/[10]`을 사용하는 쪽이 더 안전합니다.

### 글자 색상

- `/r`, `/y`와 같은 특수 문자를 사용하여 콘솔 창에 출력되는 글자 색상을 지정할 수 있습니다.

### 키보드 제어 함수

- WaitKey : 키보드의 어느 하나라도 입력할 때까지 대기합니다.
- getch : 키보드의 어느 하나라도 입력할 때까지 대기하고 입력된 값을 출력합니다.
- kbhit : 키보드 입력을 감지한 뒤 getch를 통해 입력된 값을 출력합니다.

- 키보드 입력 값을 콘솔에 출력하는 예제
- PuTTY에서 줄바꿈 문자를 쳤을 때는 `\r\n`이 보내지는 것으로 보이는데, 개발 플랫폼에 따라 `\r`이 보내질수도, `\n`이 보내질 수도 있으므로 상황에 맞게 new line 문자를 COM 포트별로 물려서 씁니다.
  ```Py
  main:
      while 1:
          if kbhit():
              ch = getch()
              if ch == 0x0d:              # 줄바꿈이 들어온 경우
                  CONSOLE << CONSOLE.ln
              elif ch == 0x08:            # Backspace 들어온 경우
                  CONSOLE << f"{ch:c}"
                  CONSOLE << " "
                  CONSOLE << f"{ch:c}"
              else:
                  CONSOLE << f"{ch:c}"
  ```






## 시리얼 포트 제어

### 시리얼 포트 기본 사양

- 시리얼 포트의 기본값은 아래와 같으며, 별다른 Configuartion 없을 시 아래 스펙으로 동작합니다.
  - 속도 : 115200 baud
  - Parity : No Parity
  - Stop Bit : 1bit
  - 전송 폭 : 8bit

### 시리얼 포트에 출력하기

- 아래 코드는 시리얼 포트를 정의하고 그 포트에 1초 간격으로 출력하는 예제입니다.

  ```Py
  COMPORT = ["COM7"]

  main:
      while 1:
          COM[0].Connect # COM7 연결, COMPORT 배열의 0번째 객체.
          COM[0] << "Hello, World!" << COM[0].ln # Hello, World! 출력.
          Delay 1000

  ```

### 시리얼 포트에 들어온 값 화면에 출력하기

- 아래 코드는 read 함수를 통해 값을 확인하는 코드입니다.
- 실시간으로 로그를 확인할 수 있는 장점이 있으나, 문자열 처리/감지가 어렵습니다.
  ```Py
  COMPORT = ["COM7"]

  main:
      COM[0].Open()
      while 1:
          if *COM[0]:
              re = COM[0].Read()
              CONSOLE << f"{re:c}"

  ```

### 시리얼 포트에 들어온 문자열 화면에 출력하기

- 아래 코드는 줄 단위로 스캔하여 화면에 출력하는 코드입니다.
- 실시간으로 로그를 확인하기 어렵지만, 문자열 처리/감지가 편합니다.
  - 실시간 감지를 위해서는 **타임아웃을 적절한 값**으로 설정하는 것이 중요합니다.
    ```Py
    COMPORT = ["COM7"]

    main:
        COM[0].Open()
        while 1:
            if *COM[0]:
                re = str
                re << COM[0]
                CONSOLE << f"{_re}"
    ```

### 시리얼 포트에 들어온 문자열 가공하기

- 아래 코드는 시리얼 코드에 `VALUE = XXX`꼴로 들어오면 XXX만 남기고 이를 숫자로 변경한 뒤 1000으로 나눈 나머지를 콘솔 창에 출력하는 예제입니다.

  ```Py
  COMPORT = ["COM7"]

  main:
      COM[0].Open()
      while 1:
          if *COM[0]:
              re = str
              re << COM[0]
              re = re - "VALUE = "
              v = atoi(re) % 1000
              CONSOLE << f"/0VAL = {v:03d}"

  ```

## 파일 입출력 제어

- 파일에 출력하기

  ```Py
  main:
      myFile = file_t
      myFile.Open("myFile.txt", "at", "\n")    # 파일 열기
      myFile << "Hello, World!" << myFile.ln   # 파일에 출력
      myFile.Close()                           # 파일 닫기
  ```

- 파일 열기는 파일명, 모드(C언어와 규격 동일), 줄바꿈 방법을 적용합니다.
  - 줄바꿈 방법은 텍스트 모드로 열 때는 `\n`만을, 바이너리 모드로 열 때는 `\r\n`으로 설정해 두면 좋습니다.
  - 로그를 계속 누적하기 위해서는 일반적으로 `at`를 모드 인자로 넣습니다.
- 파일 출력도 `<<` 연산자를 적용합니다.
- 파일을 여러개 열때는 파일명을 배열로 문자열 먼저 선언해 놓고 레퍼런스 값(_연산자)을 인수로 넣거나 아니면 각각 열거나 하면 됩니다.

## 기본 지원 구문

- `while`문
  - 앞에서 언급했다시피 `while 반복변수:`형태로 작성하면 반복변수가 참일 때 계속 반복합니다.
  - 반복변수는 되도록 int형이 들어가야 하며, int가 아닌 변수를 사용할 경우 형변환 또는 참/거짓 판별 연산을 추가합니다.
    - 예: available 연산은 참/거짓 판별 연산자 *를 추가하여 `*COM[1]` 꼴로 사용합니다.

- `if`문
  - 앞에서 본 것처럼 `if 조건:`꼴로 작성합니다.
  - 조건 변수도 int형이 되어야 하며, int형이 아닌 객체는 위에서 본것처럼 연산자를 취해 int형이 나오도록 하면 됩니다.
  - C언어와 비슷하게 `if`, `else if`, `else`를 사용해도 좋고, Python처럼 `if`, `elif`, `else`로 사용해도 좋다. 다만, 업무에 있어서 둘을 혼용하는 일은 없도록 합니다.

- `for`문
  - for문은 크게 2가지 종류로 나눌 수 있습니다.
  - C-Style for문 : for(초기값;조건;증감)의 형태로 작성합니다. 다만, 정밀한 제어가 아니라면 그리 권장되지 않습니다.
  - Simple for문 : for(변수:초기값:끝값)의 형태를 사용합니다. 다만, 정밀한 제어보다는 유한번 반복 실행할 때 쓰기 적절합니다.

- `when-case`문
  - C언어의 switch와 비슷한 구문이지만, when을 사용합니다.
  - `when 변수:` + `case 값(범위):` 뒤에 실행 구문을 넣습니다.
  - 공통 실행의 경우 `default`를 쓰는 것은 C언어와 비슷합니다.
  - `break`는 when문에서 사용하지 않습니다. case에 값을 여러개 넣는게 가능해 break 없이도 안전하게 사용이 가능하기 때문입니다. 다만, 반복문과 혼용 사용 시 break를 when문에서 사용하면 반복문을 빠져나갑니다.

- 변수 선언
  - 객체 선언 시에는 `변수 = 객체 타입`의 형태로 선언한 뒤 초기화합니다.
  - 일반 변수 선언 시에는 `변수 = 타입` 또는 `변수 = 값`으로 선언합니다. 다만, starlit 언어의 특성상 한 번 선언한 변수의 타입은 사라질 때까지 변하지 않습니다. 즉, `a = 0`이라 선언해 놓고 소수를 넣으면 a는 소수가 되지 않고 정수로 자동 형변환합니다.

## STerM의 주요 연산자

- 사칙연산
  |연산자|기능|
  |-----|---|
  |`+`|숫자의 경우 : 두 수의 합을 구합니다.<br>문자열의 경우 : 두 문자열을 이어붙입니다.|
  |`-`|숫자의 경우 : 두 수의 차를 구합니다.<br>문자열의 경우 : 특정 구문과 매칭되면 그 구문을 삭제합니다.|
  |`*`|숫자의 경우 : 두 수의 곱을 구합니다.|
  |`/`, `//`|정수의 경우 : 두 수를 나눈 몫(정수부분 이하 버림)을 구합니다.<br>소수의 경우 : 두 수를 나눈 몫을 정확히 구합니다.<br>`//` 연산자는 정수에서 나머지를 수학적으로 문제 없이 구합니다.|
  |`%`, `%%`|정수에서만 사용되며, 두 수를 나눈 나머지를 구합니다.<br>`%%` 연산자는 수학적으로 문제 없이 구합니다.|
  |`.`|객체 참조 연산자|

- 단항 연산
  |연산자|기능|
  |-----|---|
  |`-`|(정수/소수) 음의 부호|
  |`*`|(COM객체) 값이 들어온 개수 확인|
  |`int()`, `float()`...|강제 형변환 연산자, 레퍼런스의 경우 그 위치의 값을 출력.|
  |`_()`|변수의 레퍼런스를 구하는 연산자. 단독 사용시 _a꼴로 사용 가능.|
  |`len()`|배열 길이 연산자|
  |`sizeof()`|배열/변수/자료형 크기 연산자|

- 입출력 연산
  |연산자|기능|
  |-----|---|
  |`<<`, `>>`|객체로부터 변수에 값을 입력받거나 출력합니다.<br>변수쪽으로 화살표가 있으면 입력을 의미합니다.<br>객체쪽으로 화살표가 있으면 출력을 의미합니다.|

- 비교 연산
  |연산자|기능|
  |---|---|
  |`<`, `>`, `>=`, `<=`, `==`, `!=`|값을 비교하는 연산, <br>순서대로 미만, 초과, 이상, 이하, 동등, 다름을 의미합니다.<br>삼항 비교 연산이 가능합니다.(단, 자료형 매칭 필요)|
  |`&&`|두 조건을 모두 만족함을 의미합니다.|
  |`\|\|`|두 조건 중 적어도 하나 이상 만족함을 의미합니다.|
