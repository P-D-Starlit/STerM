# STerM 함수목록

- STerM 공식 지원 함수를 모아 놓은 문서.
- 중요한 함수는 볼드 처리 하였습니다.

## 목차

1) 기본 함수
1) Console 관련 함수
1) 파일 입출력 관련 함수
1) COM포트 관련 함수



## 1. 기본 함수

### 1.1. 문자열 관련 함수

- 문자열 복사 : 대입 사용 (strcpy 대응)
  - `str1 = str2` : 문자열 str1에 문자열 str2를 복사합니다.

- 문자열 이어붙이기 : 덧셈 연산 사용 (strcat 대응)
  - `str1 += str2` : 문자열 str1에 문자열 str2를 덧붙인다.

- 문자열 길이 : `*` 연산 사용 (strlen 대응)
  - `*str1` : 문자열 str1의 길이를 반환.

- 문자열 비교 : `==` 또는 `!=` 사용 (strcmp 대응)
  - `str1 == str2` : 두 문자열이 같으면 True, 다르면 False

- 특정 문구 지우기 : `-` 연산 사용
  - `str1 - "a = "` : str1 내부에 `"a = "`가 포함되어 있으면 그 내용을 통째로 지운다.
    - 2번 이상 포함되면 앞부분을 지우며, 둘 다 지우고 싶으면 두번 연산하면 된다.
    - 로그에서 특정 숫자나 문자를 감지할 때 유용하게 사용할 수 있다.

- 문자열의 10진수를 숫자로 출력 : `atoi(문자열)`
  - `atoi(str1)` : 문자열 str1의 내용을 숫자로 변환한다.
  - 주의 : 숫자가 아닌 요소가 같이 있으면 오작동할 수 있으므로 특정 문구 지우기 연산을 이용해 숫자만 남겨놓고 쓴다.
  - 소수형은 `atof`, ~~16진수는 `xtoi`를 사용한다~~.
  - STerM에서는 16진수 문자열 변환은 `strtol` 함수를 사용한다.
    - 예: `strtol("FFFFFF", NULL, 16)`

- 문자열 끊기 : string 요소 사용
  - `str1.string[5] = 0` : str1을 5자까지만 쓰고 그 이후는 끊어버린다.
  - 비슷한 예로 `str1 = str(_str2 + 5)`를 사용하면 앞부분 5자만 제외하고 복사할 수 있다.
  - 이런 원리는 로그가 가변형일 때 특정 위치만 뽑기 위해 사용한다.

- f-string : STerM은 Python의 f-string을 차용한다.  
  - f-string 사용은 Python 벤치마킹한 만큼 Python과 용법이 대부분 동일하니 Python에서 제공하는 가이드라인을 참고하면 된다.
  - f-string 포매팅한 문자열은 문자열 객체로 취급하며, 안전성 문제로 인해 고정 문자열 레퍼런스로는 동작하지 않아 일부 구형 C언어 계열 함수와 호환되지 않을 수 있다.

### 1.2. 기본 IO 함수
- Tick() : 프로그램 실행 시간 반환
- Delay(밀리초) : 밀리초 단위로 기다리기.
  - 현재 실시간 문구 안받아지는 문제 패치, sleep모드 실행은 sleep함수가 따로 있으나 sleep시간동안 데이터를 놓칠 수 있으므로 사용을 자제한다.


## 2. Console 함수

### 2.1. `CONSOLE` 객체

- 콘솔 창과 관련된 기능은 대부분 CONSOLE 객체와 연계해서 진행합니다.

- `CONSOLE.ln` : 줄바꿈을 포함하여 출력할 때는 CONSOLE.ln을 사용합니다.
  ```Py
  # 줄바꿈 포함하여 출력하기
  CONSOLE << "Hello, World!" << CONSOLE.ln
  ```

- `CONSOLE <<` : CONSOLE 객체 방향으로 `<<` 또는 `>>` 연산자 사용 시 콘솔 창에 출력함을 의미합니다.
  - 호환 가능한 자료형 : `int`, `float`, `str`, `ref`
  - `str`은 최대 256자까지 사용 가능한 문자열 객체이며, `ref`는 고정 문자열 포인터입니다.
  - f-string은 문자열 객체로 처리하며, `""` 안에 작성한 것은 고정 문자열 포인터로 취급합니다.

- `CONSOLE >>` : CONSOLE 객체에서 변수 방향으로 `<<` 또는 `>>` 연산자 사용 시 사용자로부터 값이나 문자열을 입력받습니다.
  - 호환 가능한 자료형 : `int`, `float`, `str`

### 2.2. 키보드 입력 감지

- 키보드 입력과 관련된 함수는 아래와 같습니다.(객체 없이 단독 사용)

- `kbhit()` : 키보드 입력이 들어왔는지 확인하는 함수입니다.
  - 반환 자료형은 `int`형이며, if문과 호환하여 사용합니다.

- `getch()` : 키보드 실질적 입력 문자를 반환합니다.
  - 이 함수는 문자가 입력될 때까지 기다리는 특성이 있습니다.
  - kbhit과 같이 사용하면 기다리지 않고 사용할 수 있습니다.

- `WaitKey` : 키보드가 눌릴 때까지 일시정지함을 의미합니다.
  - 실험이 끝나고 결과를 확인하고 프로그램 종료할 때 유리합니다.(로그 저장 안하는 경우)
  - 로그 파일 저장하지 않는 경우 이 구문을 사용하여 종료할 때까지 기다리지 않으면 실험 결과를 놓칠 수 있으니 유의 바랍니다.

## 3. 파일 입출력 함수

### 3.1. `FILE_Open(파일명, 모드, 줄바꿈)`

- 파일을 열고 파일 객체를 반환하는 함수입니다.
- 파일명은 일반적으로 상대 경로를 사용하며, 절대 경로 입력도 가능합니다.
- 모드는 C언어 규격과 동일하게 취급합니다.
  - `"wt"` : 텍스트 형식으로 파일을 새로 만들어 쓴다.
  - `"rt"` : 텍스트 형식으로 파일을 읽는다.
  - `"at"` : 텍스트 형식으로 파일을 이어서 붙인다.
  - `"wb"` : 바이너리 형식으로 파일을 새로 만들어 쓴다.
  - `"rb"` : 바이너리 형식으로 파일을 읽는다.
- 줄바꿈
  - 텍스트 모드 사용 시 `"\n"`으로, 바이너리 모드 사용 시 `"\r\n"` 사용을 권장합니다.

### 3.2. 파일 객체

- `FILE_Open`으로 파일 객체를 열었으면 객체와 관련된 함수를 사용하실 수 있습니다.

- `FILE.ln` : 파일에 출력 후 줄바꿈을 출력합니다.

- `FILE <<` : 파일에 출력하는 기능입니다. 문자열만 사용 가능합니다.

- `FILE >>` : 파일로부터 문자열을 입력받습니다.

- `FILE.Close` : 파일을 저장하고 닫습니다.

- `FILE.Write(1바이트)` : 1바이트를 바이너리 형태로 쓰는 함수입니다.

- `FILE.Read()` : 파일에서 1바이트를 읽습니다. EOF는 -1이 출력됩니다.

- ~~`FILE.Seek(위치)`~~ : 처음 위치 기준으로 위치를 찾는 함수입니다.(구현 예정)

### 3.3. 간편 파일 객체

- `FILE_Setup`으로 간편 파일 객체를 생성한다.
- 이렇게 세팅한 함수는 Open, Close 함수 없이 Write 함수만 가지고 파일을 누적해서 쓸 수 있다.
  - 기본 사용 예: `file = FILE_Setup("logfile.txt", "ab", "\r\n")`
  - 이렇게 정의된 함수는 파일 객체로 처리되지만, Print함수나 `<<` 연산자를 통해 파일에 쓸 수 있다.
  - 이렇게 파일을 쓰는 것이 표준으로, 매번 파일을 쓸 때마다 Open 동작과 Close 동작을 실행한다. 이렇게 하면 중간에 실행이 끊어지더라도 파일은 문제없이 누적된다.
  - 로그 파일 전부를 저장하는 테스트에서는 이걸 활용하면 편리하다.
- 간편 파일 객체로 사용하는 경우 Read 동작 등 다른 동작은 지원되지 않으니 주의.
  - 보통 로그 기록하는 용도로 파일 객체를 쓰는 STerM 특성상 읽기 기능은 잘 사용하지 않아서 그렇게 설계했다.
  - Print 함수 또는 `<<` 연산자만 정상 작동된다. 같은 출력인 Write 함수도 이렇게 사용하면 지원되지 않으니 유의.
  - 이렇게 정의된 파일을 굳이 읽거나 해야 한다면 Open함수로 열어서 읽은 후 Close 동작을 수행하면 된다.

## 4. COM포트 사용하기

### 4.1. COM포트 배열 정의

- COM포트 사용하기 위해서는 COM포트 배열을 아래와 같이 정의해야 합니다.

```Py
COMPORT = ["COM3", "COM4", "COM5", "COM6"]
```

- COM포트 배열은 보드 개수만큼 정의합니다. 나눠서 정의하면 COMPORT로 정의한 것만 유효하게 동작합니다.
- `COM3`을 제일 첫번째에 정의했으면 0번 보드가 되며, 사용할 때는 `COM[0]`을 사용합니다. `COM[3]` 사용 시 `COM6`이 동작됩니다.

### 4.2. COM포트 사용하기

- `COM[보드번호].Open()` : COM포트 연결을 시도합니다.
  - 연결 성공 시 1, 실패 시 0을 출력합니다.
  - PuTTY나 Tera Term에서 연결 중인 경우 연결 실패할 수 있으니 대상 COM포트 사용 시 반드시 다른 프로그램에서 연결 해제 후 쓰시기 바랍니다.

- `COM[보드번호] <<` : 특정 COM포트에 문자열을 출력합니다.
  - COM포트는 문자열 기반을 기본으로 제공하므로 가급적 바이너리를 직접 전송하거나 직접 받지 않습니다.
  - `COM[보드번호].ln`과 같이 사용하여 줄바꿈을 포함시킬 수 있습니다.

- ~~`COM[보드번호] >>` : COM포트로부터 전송받은 문자열을 받아옵니다.~~
  - ~~받아오는 기준은 타임아웃 초과, 줄바꿈 감지, 문자열 길이 한계 초과 등입니다.~~
  - ~~255자를 한계치로 설정하는 경우 대부분의 업무에서 문자열을 못받는 일은 없습니다.~~
  - ~~타임아웃 값에 따라 로그 인식 감도가 달라질 수 있으니 상황에 맞게 타임아웃을 조절하시기 바랍니다.~~
  - 안정성 이슈가 있어서 이 기능은 잠시 사용 중단할 예정입니다.

- `*COM[보드번호]` : COM포트로부터 대기중인 바이트를 의미합니다.
  - 이 값이 0인 경우 데이터를 받지 않습니다.
  - 이 값이 0이 아닌 경우 데이터를 받으면 됩니다.

- `COM[보드번호].Read(문자열 객체)` : 문자열 객체에 읽은 데이터를 받아옵니다. 이때, 최대 255바이트까지만 한번에 읽어들이며, Scan과 달리 줄바꿈 감지 없이 읽히는 대로 읽어 옵니다.

- `COM[보드번호].Scan(문자열 객체)` : 문자열 객체에 줄바꿈까지 읽혀진 문자열을 받아옵니다. 이때 받아오는 문자열은 줄바꿈 문자까지 포함됩니다.
  - 반환값 : 
    - `\n` 포함 시 : 양의 문자열 길이 출력, Read 동작 수행
    - `\n` 미포함 시 : 음의 문자열 길이 출력, Peek 동작 수행
    - 쉽게 말해서 양수일 때만 처리한다고 보면 됩니다.
    - 줄바꿈이 없이 출력되는 상황이라면 Scan 사용은 자제하는 것이 좋습니다.
  - Scan1 사용 시 : 줄바꿈 빼고 문자열을 받아옵니다. 정확히는 줄바꿈 포함해서 우선 받아온 후 끝에 있는 줄바꿈과 캐리지 리턴을 지우고 반환합니다.

- `COM[보드번호].Wait(시간)` : COM포트로부터 데이터를 받을 때까지 기다립니다.

- `COM[보드번호].Flush` : COM포트에 대기중인 데이터를 지웁니다. 보통 리셋 진행 시 사용합니다.

- `COM[보드번호].WriteCtrl(컨트롤과동시에누르는키보드)` : 컨트롤 + 키보드 입력 값을 전송합니다.
